壹.makefile的结构说明

makefile里面包含了五个东西：变量定义、显式规则、隐晦规则、文件指示和注释。接下来让我一一地介绍

  1. 变量的定义 在Makefile中我们要定义一系列的变量，变量一般都是字符串，这个有点像C语言中的宏，当Makefile被执行时，其中的变量都会被扩展到相应的引用位置上。
  2. 显式规则 显式规则说明了，如何生成一个或多的的目标文件。这是由Makefile的书写者明显指出，要生成的文件，文件的依赖文件，生成的命令。 刚才写的疑似shell脚本的Makefile全部都是显示规则。
  3. 隐晦规则 由于我们的make有自动推导的功能，所以隐晦的规则可以让我们比较粗糙地简略地书写Makefile，这是由make所支持的。
  4. 文件指示 其包括了三个部分，一个是在一个Makefile中引用另一个Makefile，就像C语言中的include一样。
  5. 注释 Makefile中只有行注释，和UNIX的Shell脚本一样，其注释是用“#”字符，这个就像C/C++中的“//”一样。如果你要在你的Makefile中使用“#”字符。

刚才说到了预定义变量，下面是常用的预定义变量的含义：
自定义变量

  • $*不包含扩展名的目标文件名称。
  • $+所有的依赖文件，以空格分开，并以出现的先后为序，可能包含重复的依赖文件。
  • $<第一个依赖文件的名称。
  • $?所有的依赖文件，以空格分开，这些依赖文件的修改日期比目标的创建日期晚。
  • $@目标的完整名称。
  • $^所有的依赖文件，以空格分开，不包含重复的依赖文件。
  • $% 如果目标是归档成员，则该变量表示目标的归档成员名称。

特殊变量

  • $(MAKE)：当前make解释器的文件名
  • $(MAKECMDGOALS)：命令行中指定的目标名（make的命令行参数）
  • $(MAKEFILE_LIST)：make所需要处理的makefile文件列表，当前makefile的文件名总是位于列表的最后，文件名之间以空格进行分隔
  • $(MAKE_VERSION)：当前make解释器的版本
  • $(CURDIR)：当前make解释器的工作目录
  • $(.VARIABLES)：所有已经定义的变量名列表（预定义变量和自定义变量）

gcc简介

发表于 2022-07-14

gcc 与 g++ 分别是 c & c++ 的编译器 gcc/g++ 在执行编译工作的时候，总共需要4步：

  1. 预处理,生成 .i 的文件[预处理器cpp]
  2. 将预处理后的文件转换成汇编语言, 生成文件 .s [编译器egcs]
  3. 由汇编变为目标代码(机器代码)生成 .o 的文件[汇编器as]
  4. 连接目标代码, 生成可执行程序[链接器ld]

一些参数：
-x language filename
告诉gcc filename是language所对应的语言,并且对于另一个参数之前的文件都有效，可以使用的参数有 c objective-c c-header c++ cpp-output assembler 与 assembler-with-cpp
实际用法：

-x none filename
告诉gcc 自动检测filename后缀，自动识别文件类型
实际用法：

-c
只是激活预处理，编译和汇编，也就是只是把程序做成obj文件

只激活预处理和编译，将文件变成汇编代码

将生成一个同名.S的汇编代码文件
-E
只激活预处理，这个不生成文件，需要自己重定向到一个文本文件里
实际用法：

-o
设置目标名称，默认是a.out
实际用法

-include
包含某个代码，也就是使用另一个文件作为头文件，作用相当于在代码中使用#include "filename"
实际用法：

-Idir
在使用#include "filename"的时候，gcc/g++会现在当前目录查找头文件。如果没有找到，就返回到默认的头文件当中查找。如果使用 -l定制了目录，它会现先在你指定的目录中查找，然后按照正常的顺序查找头文件。
而对于#include <filename> gcc/g++则会先在 -l指定的目录中查找，查找不到，然后到系统默认的头文件目录去查找
-dirafter dir
当在 -I指定的目录查找不到时，将会到这个目录中查找。
-I-
取消前一个参数的功能
-llibrary
指定编译的时候使用的库

-Ldir
指定编译的时候搜索库的路径。不然只在标准的目录中找
-g
只是编译器，在编译的时候输出调试信息
-static
此选项将禁止使用动态库，所以，编译出来的东西，一般都很大，也不需要什么动态连接库，就可以运行。
-share
此选项将尽量使用动态库，所以生成文件比较小，但是需要系统由动态库。
